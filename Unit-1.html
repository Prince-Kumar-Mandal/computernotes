Unit-1: Database Management System

Data

According to the Oxford Dictionary "Data is distinct pieces of information, usually formatted in a special way."
Data can be measured, collected, reported, and analyzed, and often visualized using graphs, images, or tools. Raw data may be a collection of numbers or characters before it’s been cleaned and corrected by researchers.

Information

Information is data that has been processed, structured, or organized to be meaningful and useful. It provides insights, supports decision-making, and enables problem-solving.

Features of Information

  1. Subjectivity: Influence from personal opinion or bias.
  2. Relevance: Data should be applicable and purposeful.
  3. Timeliness: Indicates when data was last updated.
  4. Accuracy: Data should be correct and verified.
  5. Correct Format: Information must be structured clearly for interpretation.

Terminologies in Database

Table, Field, Record, Tuple, Object, Keys

Table: A collection of related data in rows and columns.
Field: A single data item (column).
Record: A complete row representing one entity.
Tuple: Another term for a record.
Object: In object-oriented DBs, includes both fields and functions.
Keys: Unique identifiers like primary or foreign keys.

Data Dictionary

A centralized repository storing metadata about database structure—definitions of tables, fields, data types, and relationships.

Database Management System (DBMS)

Introduction: Software used to define, create, manage, and control access to databases.

Objectives

  • Provide data abstraction and independence.
  • Facilitate efficient data operations.
  • Ensure integrity and security.
  • Support concurrent access and recovery.

Advantages

  1. Reduced redundancy and inconsistency.
  2. Improved sharing and security.
  3. Centralized control.
  4. Enhanced integrity and recovery.

Disadvantages

  1. High setup cost.
  2. Design complexity.
  3. Performance overhead.
  4. Risk of data loss.

Types of Database Model

  • Hierarchical: Tree-like structure, parent-child relationships.
  • Network: Graph structure allowing many-to-many links.
  • Relational: Table-based format using tuples and fields.
  • Entity-Relationship (ER): Conceptual model using entities and relationships.

Integrity Constraints and Types

  • Domain: Validates data type and range.
  • Entity: Ensures unique identity via keys.
  • Referential: Maintains table relations with foreign keys.
  • Keys: Enforce uniqueness and relational links.

Normalization

Introduction

Organizing data to minimize redundancy and maintain integrity.

Normal Forms

1NF: Atomic values only; no repeating groups.
2NF: Removes partial dependency on composite keys.
3NF: Eliminates transitive dependency; non-key fields depend solely on the primary key.

Advantages

  • Reduces redundancy
  • Enhances integrity
  • Simplifies maintenance

Disadvantages

  • Complex queries
  • Performance impact due to multiple joins

Centralized and Distributed Database

Introduction

Centralized DB: Stored at one location.
Distributed DB: Spread across interconnected sites.

Advantages

  • Centralized: Easier control and management.
  • Distributed: Better reliability and access speed.

Disadvantages

  • Centralized: Single point of failure.
  • Distributed: Design complexity, synchronization challenges.

Database Security

Introduction

Database security focuses on safeguarding data from unauthorized access, manipulation, or destruction while preserving confidentiality, integrity, and availability.

Challenges

  • Preventing unauthorized access and breaches
  • Maintaining consistency and accuracy of data
  • Managing diverse user roles and privileges
  • Handling internal threats and accidental misuse

Security Measures

  • Strong authentication and role-based authorization
  • Data encryption during storage and transmission
  • Routine backups and disaster recovery planning
  • Audit trails and real-time activity monitoring

Roles of DBA (Database Administrator)

  • Installing and configuring DBMS software
  • Defining and enforcing access controls
  • Maintaining system performance and health
  • Handling data recovery and backup schedules
  • Providing support to users and developers

Explain the different models of DBMS with advantages and disadvantages.

There are several types of DBMS models, each with its own strengths and weaknesses:

  • Hierarchical Model: Organizes data in a tree-like structure.
    • Advantages: Simple to design, fast access for hierarchical data.
    • Disadvantages: Rigid structure, difficult to manage complex relationships.
  • Network Model: Allows many-to-many relationships using graph structure.
    • Advantages: More flexible than hierarchical.
    • Disadvantages: Complex design and maintenance.
  • Relational Model: Uses tables (relations) for data representation.
    • Advantages: Simple structure, powerful query capability (SQL).
    • Disadvantages: Performance can suffer with complex joins.
  • Object-Oriented Model: Stores data as objects.
    • Advantages: Good for complex data types, reusable objects.
    • Disadvantages: Less mature, limited tools/support.

What is a relational database? How is it different from other database models?

A relational database stores data in tables (relations), where each row is a record and each column is a field. It uses SQL for operations.

Difference: Unlike hierarchical or network models, relational DBs are more flexible, easier to query, and enforce integrity using keys and constraints.

What is data redundancy? How does DBMS help in reducing it?

Data redundancy means storing the same data in multiple places, which can lead to inconsistency.

A DBMS reduces redundancy by normalizing data, enforcing constraints, and using centralized storage and keys to avoid duplication.

Define normalization. Explain 1NF, 2NF and 3NF with suitable examples. Explain the normalization process with examples.

Normalization is the process of structuring a relational database to reduce redundancy and improve data integrity.

  • 1NF: All columns contain atomic values (no arrays or nested fields).
    Example: Split comma-separated phone numbers into separate rows.
  • 2NF: Remove partial dependencies (non-key attributes depend on part of a composite key).
    Example: Separate student-course table into student and enrollment tables.
  • 3NF: Remove transitive dependencies (non-key depends on another non-key).
    Example: Separate department names to another table if stored with employee data.

Explain the terms: primary key, foreign key, and candidate key with examples.

  • Primary Key: Uniquely identifies each row in a table. Example: Student ID in student table.
  • Foreign Key: Refers to primary key in another table to maintain relationship. Example: Department ID in employee table.
  • Candidate Key: Any field(s) that could be a valid primary key. Example: Email and Student ID could both uniquely identify a student.

What is SQL? Explain its components and common functions.

SQL (Structured Query Language) is a standard language used to manage and manipulate relational databases.

Components:

  • DDL (Data Definition Language): CREATE, ALTER, DROP
  • DML (Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE
  • DCL (Data Control Language): GRANT, REVOKE
  • TCL (Transaction Control Language): COMMIT, ROLLBACK

Common Functions: COUNT(), AVG(), MAX(), MIN(), SUM()

Define the following terms:

  1. Data Dictionary: A centralized repository that stores metadata—information about data structure, such as names, types, sizes, constraints, and relationships of all database objects.
  2. Primary Key: A field (or combination of fields) in a table that uniquely identifies each record. No two rows can have the same primary key, and it cannot contain NULL values.
  3. Relationship: A connection between two or more tables based on common fields. It enables data from multiple tables to be linked logically, often using keys.
  4. Data Manipulation Language (DML): A subset of SQL used to perform operations such as inserting, updating, deleting, and retrieving data from tables (e.g., SELECT, INSERT, UPDATE, DELETE).
  5. Structured Query Language (SQL): A standard programming language used to manage and manipulate relational databases. It includes components like DDL, DML, DCL, and TCL.
  6. Data Integrity: The accuracy, consistency, and reliability of data stored in a database. Integrity constraints (like primary and foreign keys) help maintain it.
  7. Data Definition Language (DDL): A subset of SQL used to define and modify database structures such as tables, indexes, and schemas (e.g., CREATE, ALTER, DROP).
  8. Data Security: Measures and protocols that protect data from unauthorized access, corruption, or theft. This includes authentication, encryption, access control, and auditing.
  9. Database System: A complete environment consisting of the database, the DBMS software, and the applications and users that interact with the system to store, modify, and retrieve data efficiently.
